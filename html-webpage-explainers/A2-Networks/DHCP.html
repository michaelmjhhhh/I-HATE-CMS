DHCP

Dynamic Host Configuration Protocol — automatically assigns IP addresses!

Computer Science Network Protocols Interactive IP Addressing
📖 Definition
DHCP
Dynamic Host Configuration Protocol

DHCP is a network protocol that automatically assigns IP addresses and other network configuration settings — such as subnet mask, default gateway, and DNS server — to devices (hosts) when they connect to a network.

Without DHCP, every device would need its IP address configured manually by a network administrator. DHCP makes this process automatic, fast, and error-free, and ensures no two devices are ever assigned the same IP address.

Key DHCP Concepts

Subnet masks, dynamic allocation & static allocation explained

🔢 What is a Subnet Mask?

A subnet mask tells a device which part of an IP address identifies the network and which part identifies the individual device (host). DHCP automatically provides the subnet mask alongside the IP address — you never have to enter it yourself.

📡 Example IP Address
192.168.1.15
This is the device's unique address on the network — like a house number on a street.
🔢 Subnet Mask
255.255.255.0
255 = that section belongs to the network. 0 = that section identifies the device. Devices sharing 192.168.1.x are on the same network.
🏘️ What It Tells Us
192.168.1.0 /24
The network address is 192.168.1.0. There are up to 254 usable device addresses (192.168.1.1–254) on this subnet.

🔄 Automatic
Dynamic Allocation

The DHCP server automatically assigns a free IP address from its pool each time a device connects. The address may be different every time — it is leased for a set period and then returned to the pool.

📱 Your phone connects to Wi-Fi → DHCP hands it 192.168.1.23 for 24 hours. Tomorrow it might get 192.168.1.47 instead.
  • ✅ No admin effort — fully automatic
  • ✅ Efficient use of the IP address pool
  • ✅ Ideal for laptops, phones, tablets
  • ⚠️ IP address can change between sessions
📌 Fixed
Static Allocation

A device is given the same IP address every single time it connects. This is configured manually on the device itself or as a DHCP reservation (the server always hands the same IP to a specific MAC address).

🖨️ The office printer is always 192.168.1.5 so every computer can find it reliably, no matter when it reboots.
  • ✅ Consistent, predictable address
  • ✅ Required for servers, printers & routers
  • ✅ Easier to apply firewall rules
  • ⚠️ More admin work; risk of human error

Types of IP Address

IPv4 and IPv6 — the two versions used on the internet today

🟠 Version 4
IPv4 — Internet Protocol Version 4

The original and most widely used version of IP addressing. An IPv4 address is made up of four numbers (0–255) separated by dots — written in dotted decimal notation. It uses 32 bits, giving about 4.3 billion possible addresses.

📡 Example: 192 . 168 . 1 . 15
  • 📏 32-bit address (4 groups of 8 bits)
  • 🌍 ~4.3 billion unique addresses
  • ⚠️ Running out — most addresses now assigned
  • 🏠 Used on most home & school networks
  • ✅ Simple, human-readable format
🟣 Version 6
IPv6 — Internet Protocol Version 6

The next generation of IP addressing, designed to replace IPv4. Written as eight groups of four hexadecimal digits separated by colons. It uses 128 bits, providing a practically unlimited number of addresses — enough for every grain of sand on Earth to have trillions.

📡 Example: 2001:0db8:85a3:0000:0000:8a2e:0370:7334
  • 📏 128-bit address (8 groups of 16 bits)
  • 🌍 340 undecillion unique addresses
  • ✅ Solves IPv4 address exhaustion
  • 🔒 Built-in security (IPsec)
  • ⚠️ More complex; not yet universal

MAC Address vs IP Address

Two very different kinds of address — both essential for networking

🔖
MAC Address
A4:C3:F0:85:AC:2D

A Media Access Control (MAC) address is a permanent, hardware-level identifier burned into a device's network card (NIC) during manufacture. It never changes — it is unique worldwide and stays the same no matter which network the device joins.

Think of it like a device's passport number — a fixed ID that belongs to the hardware forever.

🌐
IP Address
192.168.1.15

An IP (Internet Protocol) address is a logical, software-level identifier assigned by a network (often by DHCP). It can change — a device gets a different IP on each network it joins, and dynamic IPs can change each session.

Think of it like a device's home address — it tells others where to send data, but it changes if you move.

What is DHCP?

Start with a simple real-world analogy

🏨
Think of it like a Hotel Reception Desk!

When you check into a hotel, you don't bring your own room — the receptionist assigns you a room number from the available rooms. When you check out, that room becomes available for the next guest. You never have to worry about picking a room yourself.

DHCP (Dynamic Host Configuration Protocol) works exactly the same way for IP addresses! When your phone or laptop joins a network, the DHCP server automatically assigns it a unique IP address from a pool of available addresses. When you leave, the address is returned to the pool for the next device. You never have to type an IP address manually! 🎉

🤖
Fully Automatic

DHCP automatically assigns IP addresses to every device on a network — no manual setup needed. Your phone gets an IP the moment it connects to Wi-Fi.

🏊
IP Address Pool

The DHCP server maintains a pool of available IP addresses (e.g. 192.168.1.10 – 192.168.1.200) and hands them out one by one as devices connect.

⏰
Lease System

IP addresses are not given forever — they are "leased" for a set time (e.g. 24 hours). When the lease expires, the address can be reassigned to another device.

📦
More Than Just an IP

DHCP also provides the subnet mask, default gateway, and DNS server addresses — everything a device needs to communicate on the network.

🏢
Used Everywhere

Every home router, school network, and office uses DHCP. Without it, every device would need to be manually configured — imagine doing that for 500 school laptops!

🔄
D-O-R-A Process

DHCP uses a 4-step process to assign an IP: Discover → Offer → Request → Acknowledge. Remember the acronym DORA — like the explorer!

⚙️

How Does It Work?

The 4-step DORA process — how your device gets an IP address

1
📢
DISCOVER

Your device joins the network and shouts "Is there a DHCP server? I need an IP address!" — it broadcasts this to the whole network.

2
🎁
OFFER

The DHCP server replies: "Yes! I'm here. I can offer you the IP address 192.168.1.15 — it's available!" It also offers a lease time.

3
✋
REQUEST

Your device responds: "Yes please! I'd like that IP address." This formally requests the offered address from the server.

4
✅
ACKNOWLEDGE

The server confirms: "Great! IP 192.168.1.15 is now yours for 24 hours." Your device can now communicate on the network!

Advantages & Disadvantages

The good and the not-so-good

👍 Advantages
✓
Fully automatic — no manual IP configuration needed. Devices connect instantly without any user intervention.
✓
Prevents conflicts — the server ensures no two devices are ever given the same IP address, eliminating IP conflicts.
✓
Efficient address use — IPs are only assigned when devices are connected. When they leave, addresses return to the pool.
✓
Centrally managed — one DHCP server manages the whole network. Changing the gateway or DNS only needs to be done in one place.
👎 Disadvantages
✗
Single point of failure — if the DHCP server goes down, new devices can't get an IP address and can't join the network.
✗
Changing IP addresses — devices get different IPs each time they connect, making it hard to host servers or printers reliably.
✗
Security risks — a rogue (fake) DHCP server on the network could assign wrong IP settings to devices, causing a man-in-the-middle attack.
✗
Not suitable for servers — web servers, printers, and routers need a fixed IP. These must be configured manually or use a DHCP reservation.

Quick Summary

Key points to remember

📌 Definition:

A network protocol that automatically assigns IP addresses and other network settings (subnet mask, gateway, DNS) to devices when they connect to a network.

📌 The DORA Process:

Discover → Offer → Request → Acknowledge. These 4 steps happen automatically in milliseconds every time a device joins a network.

📌 Remember:

DHCP uses a lease system — IP addresses are temporary, not permanent. Devices needing a fixed IP (servers, printers) should use static IPs or DHCP reservations.
