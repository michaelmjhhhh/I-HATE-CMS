What is a Switch?

A clear definition to start with

📖
Definition

A network switch is a hardware device that connects multiple devices within a Local Area Network (LAN). It uses each device's unique MAC address to forward data frames only to the specific intended device, rather than broadcasting to all — operating at Layer 2 (Data Link Layer) of the OSI model. It builds and maintains a MAC Address Table that maps each device to a physical port.

Real-World Analogy

Connect the idea to something you already know 🧠

🎓
🎓 Think of it like a Teacher handing back marked tests!

When a teacher hands back marked tests 📄, they don't throw every paper to everyone in the room — that would be chaos! Instead, they walk directly to each student and hand them only their own paper. Private, direct, and efficient!

A Switch works exactly the same way on a network 🌐. When PC-01 wants to send data to the Printer, the switch doesn't send it to PC-02, PC-03, and the Server too — that would waste everyone's time. Instead, it checks its MAC Address Table 📋 (like a class register), finds out exactly which port the Printer is connected to, and delivers the data directly and only to that device. 🎯

Every other device on the network never even sees the data — it's like receiving a sealed envelope addressed specifically to you, and nobody else. 📬

Key Features of a Switch

What a switch can do — explained in plain English 🔍

🔧 Feature📝 What It Does — Simply Explained
Core🏷️ MAC Address Learning When a device first sends data, the switch records its MAC address and the port it is on in a MAC Address Table. It gets smarter with every packet — learning where every device lives.
Core🎯 Unicast Forwarding Once a destination MAC is in the table, the switch sends the frame only to that specific port — directly to the intended device and nobody else. This is called unicast forwarding.
Core🏫 LAN Creation Connects multiple devices (computers, printers, servers) into a single Local Area Network (LAN) — like wiring up every computer in a school lab so they can share files and resources.
Speed⚡ Full-Duplex Links Each port gets a dedicated, full-speed connection — devices can send and receive simultaneously without sharing bandwidth. A 1 Gbps switch gives every device 1 Gbps, not a share of it.
Scale🔌 Multiple Ports Switches come in many sizes: 8, 24, or 48 ports — allowing schools and offices to connect dozens of devices through one central device. Multiple switches can also be connected together.
Advanced🔒 VLANs (Managed) Managed switches support Virtual LANs (VLANs) — dividing one physical switch into separate logical networks. For example, separating student computers from teacher computers on the same hardware.

Characteristics of a Switch

The key properties that define what a switch is 🔎

🔢
Operates at Layer 2

Works at the Data Link Layer of the OSI model, using MAC addresses — not IP addresses. It does not understand routing between networks (that is the router's job).

📋
Maintains a MAC Address Table

Builds and continuously updates a table that maps every device's MAC address to a port number. This is what allows it to send data directly rather than broadcasting.

🎯
Intelligent Direct Delivery

Unlike an old hub (which broadcasts to all), a switch is smart — it reads the destination MAC address and forwards the frame only to the correct port.

🔌
Wired (Ethernet) Only

A switch works with physical ethernet cables only. It has no wireless capability — for Wi-Fi you need a separate Wireless Access Point (WAP).

🌐
Needs a Router for the Internet

A switch creates a local network (LAN) but cannot connect to the internet on its own. A router must be connected to the switch to provide internet access.

🚦
Reduces Network Congestion

By sending data only where it is needed, a switch dramatically reduces unnecessary traffic — keeping the network fast and free, even as more devices are added.

Switch vs Old Hub

Why switches completely replaced hubs 🏆

📢 Old Hub (Outdated)
✗
Sends data to ALL devices — even if only one needs it. Everyone hears everything.
✗
Wastes bandwidth — every device must process every packet, even ones not meant for it.
✗
Privacy issue — any device can potentially see all other devices' data.
✗
Slow under load — more devices = more collisions = slower network for everyone.
🎯 Modern Switch (Smart!)
✓
Sends data only to the intended device — all other devices never see it.
✓
Saves bandwidth — the network stays fast even with many devices connected.
✓
More private — data goes only to the correct destination, sealed delivery.
✓
No collisions — dedicated path per connection means full speed simultaneously.
⚖️

Advantages & Disadvantages

Every device has strengths and weaknesses 🤔

👍 Advantages
✓
🎯 Efficient bandwidth use — data goes only to the device that needs it, freeing up the network for everyone else.
✓
📈 Scalable — need more connections? Just get a switch with more ports (8, 24, 48). Multiple switches can be linked together.
✓
🔌 Plug and play — basic (unmanaged) switches work immediately with no configuration. Just plug in and go.
✓
⚡ Full speed for all — dedicated links mean every device gets its full connection speed simultaneously.
👎 Disadvantages
✗
🌐 No internet alone — a switch creates a local network (LAN) but needs a separate router to reach the internet (WAN).
✗
🔌 Wired only — switches only support ethernet cables. Wi-Fi devices need a separate Wireless Access Point (WAP).
✗
🛡️ Basic security on unmanaged models — simple switches have no security features. Advanced features (VLANs, monitoring) require expensive managed switches.
✗
💰 Cost increases with features — managed switches with VLANs, QoS, and security monitoring are significantly more expensive than basic models.

Quick Summary

Key points to remember for your exam 🎓

📌 Definition

A device that connects multiple devices in a LAN, using MAC addresses to send data frames only to the intended recipient. Operates at Layer 2 (Data Link Layer).

📌 Key Difference from Hub

A hub broadcasts to ALL devices — wasteful and slow. A switch is smart — it uses MAC addresses to send data ONLY to the specific intended device.

📌 Remember

Switch = LAN only. Wired only. Needs a router for internet. Needs a WAP for Wi-Fi. Maintains a MAC Address Table to know which device is on which port.
